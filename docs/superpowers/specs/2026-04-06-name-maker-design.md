# 작명 웹사이트 디자인 스펙

**날짜:** 2026-04-06
**프로젝트:** name-maker
**범위:** 이름 생성 (감정 기능 제외, 추후 확장 가능)

---

## 개요

사용자가 생년월일·성씨를 입력하면 사주팔자를 계산하고, 전통 작명 이론(수리성명학·오행성명학·발음오행·용신성명학·음양배합·자의)을 모두 반영한 한자 이름 TOP 5를 추천하는 웹앱.

- 외부 API 없음 — 만세력 테이블, 인명용 한자 DB, 81수리 테이블 모두 내장
- React/Vite + TypeScript + Zustand
- 라이트 테마, 골드(#C9A84C) 포인트 컬러

## 핵심 로직

### 이름 생성 필터링 순서

```
1단계 [필수]: positive=true인 한자만 사용
2단계 [성별]: gender 필터
3단계 [자연스러움]: 반복 음절 없음 + 받침+초성 충돌 없음 (항상 적용)
4단계 [수리]: 4격 모두 81수리 길격
5단계 [용신]: 자원오행에 용신 포함
6단계 [자원오행]: 상생 구조
7단계 [발음오행]: 상생 구조
8단계 [음양]: 홀짝 배합 균형

→ 결과 미달 시 8→7→6→5단계 완화, 점수제 폴백
→ 1~3단계는 절대 완화 안 함
```

### 수리 4격
- 원격: 이름 첫째 자 획수
- 형격: 성 + 이름 첫째 자
- 이격: 이름 전체 획수 합
- 정격: 성 + 이름 전체 / 81 초과 시 나머지

### 발음오행 매핑
- 木: ㄱ, ㅋ
- 火: ㄴ, ㄷ, ㄹ, ㅌ
- 土: ㅇ, ㅎ
- 金: ㅅ, ㅈ, ㅊ
- 水: ㅁ, ㅂ, ㅍ

## UI/UX

### 입력 화면
1. 생년월일 (날짜 피커)
2. 태어난 시간 (선택, "모름" 포함)
3. 성별: **남아 / 여아 / 성별 무관**
4. 성씨 입력 → 흔한 성 자동 매칭, 목록 선택
5. 이름 글자 수: 1자 / 2자
6. "이름 생성" 버튼

### 결과 화면
- 상단: 사주팔자 + 용신 오행 강조
- TOP 5 이름 카드
  - 기본: 한자 크게 + 독음 + 뜻 + 이론별 뱃지 6개
  - 펼침: 수리 4격 표, 오행 흐름, 발음오행, 음양

### 디자인 시스템
- 배경: #FAFAF8 (따뜻한 흰색)
- 포인트: #C9A84C (골드)
- 오행 색상: 木=초록, 火=빨강, 土=노랑, 金=회색, 水=파랑
- 폰트: Noto Serif KR (한자), Pretendard (UI)
